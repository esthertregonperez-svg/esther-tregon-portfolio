# 🏗️ 01-ConstructionDB

## Base de datos MySQL para la gestión de una empresa de construcción

ConstructionDB es una base de datos relacional desarrollada en **MySQL** diseñada para centralizar y gestionar la información de una empresa del sector de la construcción.

El proyecto integra en una única base de datos la gestión de **clientes, obras, presupuestos, empleados, materiales, proveedores, pedidos de compra, control de stock y facturación**, proporcionando una estructura sólida, normalizada y preparada para futuras ampliaciones mediante aplicaciones web y sistemas ERP.

El objetivo principal es sustituir procesos manuales y hojas de cálculo por un sistema estructurado que garantice la integridad de los datos, optimice la organización de la información y facilite la toma de decisiones dentro de la empresa.

Durante el desarrollo se aplicaron principios de modelado **Entidad-Relación (E/R)**, normalización de datos, integridad referencial y diseño relacional, implementando además consultas SQL, vistas y una arquitectura preparada para incorporar automatizaciones, procedimientos almacenados, triggers y nuevas funcionalidades.

---

# 🎯 Objetivos

- Diseñar una base de datos relacional normalizada.
- Centralizar toda la información de la empresa.
- Mejorar la organización y trazabilidad de los datos.
- Reducir errores derivados de procesos manuales.
- Facilitar consultas e informes mediante SQL.
- Crear una base sólida para el desarrollo de futuras aplicaciones empresariales.

# 📊 Estructura de la base de datos

Entre las principales entidades se encuentran:

- Clientes
- Obras
- Presupuestos
- Facturas
- Empleados
- Materiales
- Stock
- Proveedores
- Pedidos de compra

Además, incorpora vistas SQL para facilitar el acceso a la información y mejorar el rendimiento de consultas frecuentes.
